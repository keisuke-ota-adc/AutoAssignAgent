# 稼働割当ステータス

稼働割当（Transaction）のステータスを定義する。大分類は**稼働待ち・稼働終了待ち・稼働終了・無効**の4区分とする。プロジェクトや予約の大分類が「～待ち」形式であるのに対し、稼働割当は**ライフサイクル上の段階**をそのまま示す語を用いる。

## 1. 目的

- 予約の**小分類：予約完了**（確定待ち）到達後に生成される**稼働コミット**（誰が・いつからいつまで・どのプロジェクトか）を一覧・レポートで共通言語で扱う。
- 「次に誰が何を待つか」よりも、**いまその枠がどの段階か**（まだ始まっていない／動いている／片付いた／無かったことにする）を直感的に示す。

## 2. データモデル上の前提

- 稼働割当は Transaction（Tr）として扱う（`docs/data-model-flowchart.md`）。
- **予約Tr**が**同種の情報を複数内包**し、**予約・大分類：確定待ち・小分類：予約完了**に至ったタイミングで、対応する稼働割当Trが**生成**される（`docs/reservation-status.md`）。業務上アクティブなコミットは、生成後は稼働割当側が中心になる想定。
- 確定後の体制変更・欠員・別人配置などは、原則**稼働割当の更新**や**別予約**で表す（プロジェクト大分類は「完了待ちのまま」など、`docs/project-status.md` と整合）。

## 3. 大分類（確定）

順序はライフサイクルの目安であり、厳密な必須順序は業務ルールで別途定義する。

| 順 | 表示名 | 意味（目安） |
|----|--------|----------------|
| 1 | 稼働待ち | 期間の**開始前**。枠はあるが、まだ現場稼働に入っていない。 |
| 2 | 稼働終了待ち | **稼働期間中**の帯。勤怠・承認・引き継ぎなど**事務上の締め**が残る場合も、大分類ではここに含め、細部は詳細で表す。 |
| 3 | 稼働終了 | その稼働割当としての**役目は完了**。参照・履歴が主。 |
| 4 | 無効 | **打ち切し・取消・誤登録**など。成立しなかった、または巻き戻したコミット。 |

### 3.1 稼働待ち

- 先行予約により、プロジェクトがまだ**受注・要因確定待ち**の段階でも存在しうる。
- 開始日を迎えたタイミングで**稼働終了待ち**へ進む。申請時点で開始日が過去に近い場合は、生成直後から**稼働終了待ち**とするなど、実装で単純化してよい。

### 3.2 稼働終了待ち

- プロジェクトの**完了待ち**と期間が重なりやすいが、**1対1ではない**（プロジェクトが続いていても、個別の稼働割当は先に**稼働終了**しうる）。
- 欠員・代理・付け替えの内訳は**詳細**（将来定義）や関連レコードで表す。

### 3.3 稼働終了

- 期間満了・成果完了・合意による早期終了など、ビジネス上**正常に閉じた**結果。
- 事務上の締めを**稼働終了**に入れるか**稼働終了待ち**に含め続けるかは運用で選択可能。本書の大分類では、締めも含め**稼働終了待ち**に載せ、すべて済んだら**稼働終了**とする想定をデフォルトとする。

### 3.4 無効

- プロジェクトの**失注・中止**に伴う整理、予約の**不成立**後の巻き戻し、誤操作の取消など。
- 予約側の**不成立済み**（解放・却下）とセットで発生しうる。

### 3.5 フローチャート（大分類・詳細）

本流は「稼働待ち→稼働終了待ち→稼働終了」。大分類に**無効**もあり、本流を経ずに遷移しうる。subgraph のタイトルが**大分類**、内側のノードが**詳細**。大分類「稼働終了」と小分類「稼働終了」は同名のため、画面では接頭語などで区別してよい。点線は無効への遷移の例。同図は `docs/data-model-flowchart.md` にも掲載する。

```mermaid

  subgraph w1["大分類: 稼働待ち"]
    direction TB
    w1a[開始日待ち]
    w1b[準備中]
  end
  subgraph w2["大分類: 稼働終了待ち"]

    w2a[稼働中]
    w2b[更新確認中]
  end
  subgraph w3["大分類: 稼働終了"]

  end

  end

  w1 -.-> w4
  w2 -.-> w4
```

## 4. 予約・プロジェクトとの対応（要約）

| 観点 | 内容 |
|------|------|
| 予約 | **確定待ち・予約完了**で稼働割当を生成。**確定済み**は顧客合意など予約Trの後続。 |
| 稼働割当 | 生成直後は通常**稼働待ち**（**開始日待ち**／**準備中**）。開始後は**稼働終了待ち**、完了後は**稼働終了**。 |
| プロジェクト | 大分類は**受注・要因確定待ち**・完了待ち・クローズ待ちの粗い帯。稼働割当の**稼働待ち・稼働終了待ち・稼働終了**と期間で対応するが、必ずしも同じタイミングで進まない。 |
| 変更 | 確定後の価格・要因は主にプロジェクトと見積。**人と期間のコミット**の変更は稼働割当の更新や新規予約が本体。 |

## 5. 詳細について

- 各大分類の**小分類**は §3.5 の図に従う。**無効**の詳細は必要になったときに定義する。
- 欠員補填・代理稼働・期間変更承認待ちなど、追加の詳細が必要になったときは本書を拡張する。
- 命名は画面やキューで**次の行動が読み取りやすい**ことを優先できる（予約ステータスの詳細と同趣旨）。
